## 2. 서비스 개요

### 2.1 서비스명
- 마켓인포 (Market Info)

### 2.2 서비스 정의
- 전통시장 상품 정보와 점포 콘텐츠를 디지털로 제공하고, AI를 활용한 정보 생성/운영 보조를 통해 소비자의 장보기 행동을 전통시장 방문으로 전환하는 서비스.
- 홈 화면은 상품 판매 피드가 아닌 방문 유도 허브로 운영하며, 상품 상세 정보는 검색/상세 화면에서 제공한다.

### 2.3 목표
- 소비자: 방문 전 의사결정 가능 수준의 정보 제공, 장보기 효율 향상, 경험 가치 인지.
- 상인: 입력 부담 최소화, 노출 증가, 가격/재고 운영 효율 개선.
- 플랫폼: 유입 → 소비 확장 → 재방문의 선순환 구조 형성.

## 3. 범위

### 3.1 인스코프 (MVP)
- 소비자용

1. 상품 검색/비교
2. 드랍(입고 알림)
3. 방문 유도형 홈 허브(드랍/행사/점포 스포트라이트/퀵 액션)
4. 점포 위치/추천 동선

> 전통시장 지도에서 확인하려면 zoom으로 전통시장 크기에 맞게 설정하는 것 필요
(dynamic -> 해당 전통시장의 경로 데이터만 db에서 조회하여 화면에 표시)
> 내 위치 정보를 받고, 근처 xM를 설정하여 해당 영역까지만 정보가 조회되도록 제한

- 상인용
1. AI 보조 상품 정보 등록(사진/음성 기반)
    -> 등록되는 절차 구체화
2. AI 가격 정책 보조
    -> 시세 api db화 하여 조회 시 카테고리로 연결해서 가져올 수 있게 처리
3. 재고/판매 관리
    -> 예정입고 관리 (입고일시, 수량 등록)

4. 상인 스토리 생성

- 공통
1. 사용자 인증 및 권한관리(소비자/상인)
2. 알림 시스템(앱 푸시/인앱)

## 5. 기능 요구사항

## 5.1 소비자 기능

### FR-C-01 점포, 상품 검색 및 비교
- 목적: 시장 방문 전 구매 의사결정 지원
- 설명: 상품명 검색 시 점포별 상품 정보 비교 제공
- 입력: 상품명 키워드, 필터(가격대/상태/시장)
- 출력: 점포명, 가격, 상품 상태, 이미지, 업데이트 시각
- 처리 규칙
    1. 동일 상품군은 가격 오름차순 기본 정렬
    2. 데이터 최신성 기준(예: N시간 이내) 미충족 시 "갱신 필요" 표시
    3. 품절 점포는 기본 숨김(필터로 보기 가능)
- 예외
    1. 검색 결과 없음: 유사 상품 추천
    2. 데이터 불일치: 최근 업데이트 기준 우선 노출
    3. 일시적 오류 발생 시 재시도 및 고객센터 문의 경로 안내
- 수용 기준(AC)
    1. 키워드 검색 2초 이내(평균) 결과 반환
    2. 최소 1개 이상 비교 항목(가격/상태/이미지) 표시
- 노출 정책
    1. 본 기능은 홈 화면의 상품 나열을 대체하지 않으며 검색 진입 후에만 상세 상품 비교를 제공

### FR-C-02 드랍(입고 알림)
- 목적: 방문 트리거 생성
- 설명: 특정 상품의 예정 입고를 이벤트로 노출, 사용자가 알림 구독
- 입력: 상품/점포/시장 선택, 알림 시간 옵션
- 출력: 입고 일정 카드, 리마인드 알림
- 처리 규칙
1. 예정 시각 기준 T-24h/T-1h 알림(기본값, 추후 설정 가능)
2. 실제 입고 확정 시 상태를 "확정"으로 변경
- 예외
1. 입고 취소/지연 시 즉시 정정 알림 발송
- 수용 기준
1. 사용자가 알림 설정 후 구독 상태를 즉시 확인 가능

### FR-C-03 장보기 에이전트 (RAG)
- 목적: 장보기 행동의 시장 전환
- 설명: 식단/요리 의도 입력 시 메뉴 제안 + 장보기 리스트 생성 + 시장 상품 매칭
- 입력: 자연어 질문(예: "오늘 저녁 뭐 먹지?"), 인원/예산/선호(선택)
- 출력: 추천 메뉴, 재료 리스트, 점포 매칭 결과
- 처리 규칙
1. 레시피 근거 데이터(RAG) 기반 응답
2. 재료별 대체품/계절품 선택지 제공
3. 품절 시 인근 대체 점포 제안
- 예외
1. 질의 모호: 보완 질문 1회 이내 요청
2. 매칭 실패: 일반 장보기 리스트만 제공
3. 일시적 오류 발생 시 기본 추천 템플릿과 재시도 안내 제공
- 수용 기준
1. 메뉴/리스트/매칭 3단계 결과가 하나의 플로우에서 제공됨
2. 추천 사유(간단 근거) 표시
- RAG 데이터 소스
1. MVP: 샘플 레시피 데이터셋 사용
2. 실서비스: 외부 레시피 DB 및 사용자 데이터 기반 확장

### FR-C-04 방문 유도형 홈 허브 (일일 피드)
- 목적: 소비자의 시장 방문 계기 생성
- 설명: 드랍, 시장 행사, 점포 스포트라이트, 방문 유도 퀵 액션을 홈에서 제공
- 입력: 사용자 관심사, 위치(선택), 방문 시장
- 출력: 홈 모듈(드랍 히어로, 행사 카드, 점포 스포트라이트, 퀵 액션 버튼)
- 처리 규칙
1. 홈 상단의 상품 노출은 드랍 카드 중심으로 제한
2. 일반 상품 가격/상세 정보는 검색·상세 화면으로 유도
3. 일 1회 이상 피드 갱신
4. 신규/인기/마감임박 태그 표시
- 예외
1. 당일 데이터 부족 시 주간 인기 콘텐츠 대체 노출
2. 피드 로딩 실패 시 기본 카테고리 카드와 재시도 버튼 제공
- 수용 기준
1. 홈 진입 시 방문 유도 모듈(드랍/행사/점포)이 1개 이상 노출
2. 홈의 주요 액션 버튼(시장 찾기/입고 알림/정보 가이드봇/추천 동선) 동작
3. 홈에서 점포 상세 및 드랍 상세로 이동 가능

### FR-C-05 점포 위치 및 추천 동선
- 목적: 체류시간 증가 및 소비 확장
- 설명: 장보기 리스트 기반 점포 위치 안내와 최적 동선 추천
- 입력: 선택 상품/점포, 현재 위치(선택)
- 출력: 점포 지도, 이동 경로, 예상 소요시간
- 처리 규칙
1. 최소 이동 거리 기준 기본 경로 생성
2. 필수 구매 항목 우선 방문 순서 배치
- 예외
1. 지도 데이터 누락 점포는 목록형 안내로 대체
2. 점포 정보 부족 시 인근 시장 안내 제공
- 수용 기준
1. 경로 재계산 3초 이내
2. 경유 점포별 체크 상태 제공
- 지도 데이터 한계 및 대응
1. 실내 GPS 정확도 한계를 고려
2. 점포 위치는 구역 기반(텍스트) 안내를 병행 제공
3. 지도 미지원 점포는 리스트 형태로 안내

## 5.2 상인 기능

### FR-M-01 상품 정보 등록 (AI 보조)
- 목적: 디지털 입력 부담 최소화
- 설명: 상인이 사진/음성/간단 텍스트 입력 시 상품정보 자동 구조화
- 입력: 상품 사진, 음성, 최소 필수항목(가격/재고)
- 출력: 표준 상품 카드(명칭/분류/설명/가격/재고)
- AI 입출력 정의
1. 입력: 이미지, 음성 텍스트
2. 출력: 상품명, 카테고리, 설명
- 처리 규칙
1. AI 추출 결과는 게시 전 상인 최종 확인 필수
2. 필수값 누락 시 저장 불가
- 예외
1. 인식 실패 시 수동 입력 폼 제공
2. 일시적 오류 발생 시 임시 저장 및 재시도 안내
- 수용 기준
1. 입력 시작부터 게시 완료까지 60초 이내(평균)

### FR-M-02 가격 정책 보조 (AI)
- 목적: 가격 신뢰성 및 운영 효율 확보
- 설명: 유사 상품/시장 상황 기반 권장 가격 제안
- 입력: 상품 분류
- 출력: 권장가 범위, 할인 제안, 사유
- 처리 규칙
1. 제안은 권고이며 최종 가격 결정권은 상인에게 있음
- 예외
1. 관련 카테고리 시세 없을 시 제안하지 않음

### FR-M-03 재고 및 판매 관리 보조
- 목적: 회전율 개선
- 설명: 재고 상태를 분석해 할인/마감/진열 우선순위 제안
- 입력: 현재 재고, 입고일, 판매 속도
- 출력: 운영 액션 제안 리스트
- 처리 규칙
1. 임계치(재고일수/유통기한)에 따라 우선순위 자동 산정
2. 액션 실행 여부는 상인 선택
- 예외
1. 판매이력 부족 시 보수 전략만 제안
2. 분석 실패 시 기본 재고 체크리스트 제공
- 수용 기준
1. 유통기한 D-2 이하 또는 재고 10% 미만 시 위험 표시

### FR-M-04 상인 스토리 생성
- 목적: 점포 차별화 콘텐츠 확보
- 설명: 상인 이력/철학/대표상품 기반 스토리 자동 생성
- 입력: 인터뷰 문답, 점포 키워드, 사진
- 출력: 점포 소개문, 카드형 스토리, 해시태그
- 처리 규칙
1. 톤앤매너 템플릿 선택 가능
2. 게시 전 편집/승인 단계 제공
- 예외
1. 민감정보 포함 시 자동 마스킹/수정 요청
2. 생성 실패 시 인터뷰 입력값 기반 기본 소개문 템플릿 제공
- 수용 기준
1. 3가지 길이(짧음/보통/상세) 버전 생성

## 6. 데이터 생성 및 관리 주체

#### ① MVP(시연용)
- 상품/점포/드랍 데이터: 팀이 시나리오 기반으로 직접 생성한 모킹 데이터 사용
- 가격 및 재고 정보: 고정값 또는 단순 상태값(있음/적음/품절)으로 설정
- 레시피 데이터: 샘플 DB 기반
- 드랍 이벤트: 수동 등록

#### ② 실서비스(확장 시)
- 상품 데이터
1. 생성: 상인
2. 보정: AI 자동 정형화
3. 검증: 상인 최종 승인
- 드랍
1. 생성: 상인
2. 알림: 시스템 자동 발송
- 가격
1. 제안: AI
2. 결정: 상인
- 재고
1. 입력: 상인
2. 상태 관리: 시스템

한 줄 핵심 문장
“시연 단계에서는 모킹 데이터를 활용하되, 실제 서비스에서는 상인 입력과 AI 보조 구조로 전환 가능하도록 설계하였다.”

## 7. 사용자 시나리오 (요약)

### 7.1 소비자 장보기 시나리오
1. 사용자가 홈에서 드랍/행사/점포 스포트라이트 확인
2. 홈 퀵 액션(시장 찾기/입고 알림/정보 가이드봇/추천 동선) 중 하나 선택
3. 필요한 경우 검색으로 이동해 점포별 상품 비교
4. 원하는 상품 드랍 알림 설정
5. 추천 동선 따라 시장 방문

### 7.2 상인 등록 시나리오
1. 상인이 상품 사진/음성 업로드
2. AI가 상품 카드 자동 생성
3. 상인이 가격/재고 검토 후 게시
4. 재고 임계치 도달 시 할인 제안 수신

### 7.3 핵심 사용자 플로우

#### 플로우 1: 장보기 유입
1. 홈에서 드랍/행사 확인
2. 점포 스포트라이트 확인
3. 액션 버튼 선택(검색/알림/동선)
4. 방문 결정

#### 플로우 2: 장보기 확장
1. 장보기 리스트 생성

4. 소비 확장

핵심 문장
“본 서비스는 검색 → 비교 → 방문 → 소비 확장의 흐름을 중심으로 설계되었다.”

## 8. 수용 기준 (통합)

- 검색/비교, 드랍알림, 방문 유도형 홈 허브, 동선추천, 상인 등록이 E2E로 동작
- 상인 입력 프로세스 평균 작업시간 60초 이내
- 소비자 플로우에서 검색→리스트→동선 이동 연계 가능
- 홈은 방문 유도 중심으로 동작하며 상품 상세 노출은 드랍 카드 외 최소화

## 9. 리스크 및 대응

- 데이터 최신성 저하: 갱신 SLA 및 "갱신 시각" 노출
- 상인 디지털 적응 격차: 음성/사진 중심 UX, 운영 교육 최소화
- AI 오추천/환각: 근거 표시, 최종 확인 단계, 신고/수정 루프
- 초기 데이터 부족: 인기시장/핵심 품목 중심 단계적 확장

## 10. 일정/릴리스 (초안)

- Phase 1 (MVP): 방문 유도형 홈 허브, 검색/비교, 드랍 알림, 동선 추천, 상인 등록
- Phase 2: RAG 장보기 에이전트, 가격/재고 고도화

## 11. 확인 요청 사항 (PDF 미기재)

다음 항목은 구현 전 확정이 필요합니다.

1. 플랫폼 범위: 모바일 앱(iOS/Android)
2. 초기 대상 시장: 런칭 지역/시장 수(예: 서울 N개 시장)
3. 사용자 인증 방식: 소셜로그인, 게스트 허용
4. 상인 인증 방식: 사업자/점포 확인 절차와 승인 주체
5. 알림 채널 우선순위: 푸시 (FCM)
6. 지도/위치 정책: 실내 지도 정밀도 및 GPS 오차 허용범위
7. 데이터 갱신 주기: 가격/재고/카탈로그 갱신 SLA
8. AI 정책: 금칙어/안전 가이드/환각 대응 수준
9. 개인정보/보관정책: 보관기간, 파기정책, 제3자 제공 여부
10. 운영 주체: 지자체/시장상인회/운영사 역할 분담
11. 다국어 필요 여부: 관광객 대상 영어/중국어 지원 (상인 스토리 생성 시 번역하여 앱 사용자에 맞춰 다국어로 제공)

## 12. 용어 정의

- 드랍(Drop): 특정 상품의 예정 입고 이벤트
- 장보기 에이전트: 사용자 의도 기반 메뉴/리스트/상품 매칭 AI 기능
- 카탈로그: 시장의 일일 할인/행사/추천 정보를 제공하는 피드
- 시장 전환: 온라인 탐색 행동이 오프라인 시장 방문/구매로 이어지는 상태
