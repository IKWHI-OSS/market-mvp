## 2. 문서 목적 및 범위

### 2.1 목적
- 기능명세서의 요구사항을 실제 화면 단위로 구체화하여, 기획/디자인/프론트엔드/백엔드/QA가 동일한 기준으로 개발 가능하도록 한다.

### 2.2 범위
- 플랫폼: 모바일 앱 우선(iOS/Android 공통 UX 기준)
- 포함 범위
1. 소비자 앱 화면
2. 상인 콘솔(모바일 우선)
3. 공통 인증/알림/예외 화면
- 제외 범위
1. 비주얼 스타일가이드(색상, 타이포 상세 토큰)
2. 브랜드 그래픽 산출물
3. 개발 프레임워크별 구현 코드

### 2.3 릴리스 범위
- MVP
1. 상품 검색/비교
2. 드랍(입고 알림)
3. 방문 유도형 홈 허브(드랍/행사/점포 스포트라이트)
4. 점포 위치/추천 동선
5. 상인 상품 등록(AI 보조)
6. 공통 인증/알림
7. 장보기 에이전트(RAG)
- Phase 2
1. 고도화 운영 보조(가격/재고)
2. 장보기 에이전트(RAG) 고도화

## 3. 사용자 및 접근 권한

### 3.1 사용자 유형
- 소비자(User)
- 상인(Merchant)

### 3.2 접근 정책
- 역할 기반 접근 제어(RBAC)
- 역할별 기본 홈 분기
1. 소비자: 방문 유도형 홈 허브
2. 상인: 점포 운영 홈

## 4. UX 원칙

1. 방문 전 의사결정: 가격/재고/입고/위치 정보를 빠르게 확인할 수 있어야 한다.
2. 최소 입력: 상인 입력은 사진/음성 중심으로 60초 내 완료를 목표로 한다.
3. 상태 가시성: 모든 핵심 액션은 상태값(요청, 확인, 준비 등)을 즉시 확인 가능해야 한다.
4. 실패 대응 일관성: 오류 발생 시 재시도/대체경로/문의 경로를 일관된 패턴으로 제공한다.
5. 현실 데이터 표시: 최종 업데이트 시각과 데이터 변동 가능성을 항상 노출한다.

## 9. 화면별 상세 명세

## 9.1 SCR-C-01 소비자 홈(방문 유도 허브)
- 목적: 소비자의 시장 방문 계기를 우선 제공
- 진입 경로: 앱 실행 후 기본 랜딩
- 핵심 컴포넌트
1. 상단 검색바
2. 드랍 히어로 카드(캐러셀)
3. 방문 유도 퀵 액션(시장찾기/입고알림/정보가이드봇/추천동선)
4. 행사 카드 섹션
5. 점포 스포트라이트 카드
- 인터랙션
1. 드랍 카드 탭 시 상세 이동
2. 퀵 액션 버튼 탭 시 기능 화면 이동
- 상태
1. 로딩: 스켈레톤 카드
2. 빈 상태: 인기 콘텐츠 대체 노출
3. 오류: 재시도 + 기본 홈 모듈 카드
- 데이터 표시 규칙
1. 홈의 상품 정보는 드랍 카드 외 최소 노출
2. 일반 상품 상세 정보는 검색 화면으로 유도
3. 신규/인기/마감임박 태그 노출
4. SLA 벗어난 데이터는 "갱신 필요" 배지
- 수용 기준
1. 홈 진입 후 방문 유도 핵심 모듈(드랍/행사/점포) 2초 이내 표시
- CTA (Primary Actions)
1. 드랍 보기 → SCR-C-04 이동
2. 시장 행사 보기 → SCR-C-10 이동
- Navigation (Next Screens)
1. 드랍 카드 클릭 → SCR-C-04
2. 점포 스포트라이트 클릭 → SCR-C-11

## 9.7 SCR-C-07 지도/추천 동선
- 목적: 이동 효율화와 체류 시간 증가
- 핵심 컴포넌트
1. 지도 뷰
2. 경로 요약(거리, 시간)
3. 경유 점포 리스트

1. 실내 GPS 정확도 한계 안내
2. 구역 기반 텍스트 안내 병행
3. 미지원 점포는 리스트형 안내
- 상태
1. 점포 정보 부족: 인근 시장 안내
2. 경로 실패: 기본 직선 경로 + 리스트 안내
